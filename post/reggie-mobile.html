为什么用到AtomicInteger是因为AtomicInteger 是原子多线程int基本数据类型操作中不会出现多线程赋值失败错误,因为其原子性所以这里很关键

AtomicInteger 类提供了一组丰富的方法来执行原子操作，确保在多线程环境中对整数的修改是安全的。以下是 AtomicInteger 的一些常用方法：

  1. 基本原子操作：

    • int get()：获取当前值。
    • void set(int newValue)：设置当前值为 newValue。
    • int getAndSet(int newValue)：设置当前值为 newValue 并返回旧值。

  2. 增加和减少：

    • int incrementAndGet()：原子地将当前值增加 1，并返回增加后的值。
    • int getAndIncrement()：原子地将当前值增加 1，并返回增加前的值。
    • int decrementAndGet()：原子地将当前值减少 1，并返回减少后的值。
    • int getAndDecrement()：原子地将当前值减少 1，并返回减少前的值.

  3. 条件更新：

    • boolean compareAndSet(int expect, int update)：如果当前值等于 expect 则原子地设置为 update 并返回 true，否则不修改并返回 false。
    • boolean weakCompareAndSet(int expect, int update)：与 compareAndSet 类似，但该方法提供了更少的保证，可能在某些情况下循环使用。

  4. 累加和更新：

    • int addAndGet(int delta)：原子地将 delta（增量）加到当前值上，并返回新的值。
    • int getAndAdd(int delta)：原子地将 delta（增量）加到当前值上，并返回增加前的值。

  5. 更新方法：

    • int updateAndGet(IntUnaryOperator updateFunction)：原子地应用 updateFunction 函数到当前值上，并返回更新后的值。
    • int getAndUpdate(IntUnaryOperator updateFunction)：原子地应用 updateFunction 函数到当前值上，并返回更新前的值。

  6. 懒惰设置：

    • boolean compareAndExchange(int expect, int update)：如果当前值等于 expect，则原子地设置为 update 并返回 true。否则，保持原值。
    • boolean compareAndExchangeAcquire(int expect, int update)：与 compareAndExchange 类似，但提供了额外的内存一致性保证。
    • boolean compareAndExchangeRelease(int expect, int update)：与 compareAndExchange 类似，但具有不同的内存一致性效果。

这些方法使得 AtomicInteger 成为在多线程环境中进行线程安全操作的强大工具。您可以根据需要选择适当的方法来实现您的逻辑。
