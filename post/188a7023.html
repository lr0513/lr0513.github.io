python内存管理

Python采用基于值的自动内存管理模式，变量并不直接存储值，而是存储值的引用或内存地址

  • 基于值的模式：
    小整数、短字符串等会驻留内存，同值变量指向同一地址。
  • 变量赋值：
    变量存储值的内存地址（引用），赋值改变指向。
  • 与 Java 对比：
    Java 对象变量也是引用传递，类似 Python。

  1. while 循环
  • 注意事项：

      • continue 和 break 在 for 和 while 循环中使用一致。
      • 在嵌套循 环中，只能作用在所在的循环上，无法对上层循环起作用。

while VS for

while 需要手动控制循环变量的变化（比如index += 1），否则可能会导致无限循环。
for不需要显式地管理循环变量，因为它会自动遍历可迭代对象。

  • 在循环控制上：
    • while循环可以自定循环条件，并自行控制
    • for循环不可以自定循环条件，只可以一个个从容器内取出数据
  • 在无限循环上：
    • while循环可以通过条件控制做到无限循环
    • for循环理论上不可以，因为被遍历的容器容量不是无限的
  • 在使用场景上：
    • while循环适用于任何想要循环的场景
    • for循环适用于，遍历数据容器的场景或简单的固定次数循环场景

序列的切片操作
切片是指从一个序列中获取一个子序列的操作，序列支持切片，允许通过指定起始位置、结束位置和步长来截取部分元素。
语法：序列[起始下标:结束下标:步长]

  • 起始下标：表示从何处开始截取，若省略，则默认为从序列的头部开始（索引 0）。
  • 结束下标：表示在哪个位置结束截取，但不包括结束位置本身，若省略，则默认为序列的结尾。
  • 步长：表示每次截取元素的间隔，默认为 1。可以为正数，也可以为负数（用于倒序截取）。
  1. 基本示例：从序列中取出部分元素。
  2. 省略起始和结束下标：
    • 省略起始下标：默认从序列的开始位置截取。
    • 省略结束下标：默认截取到序列的结尾。
  3. 使用步长：
    • 步长 1：每次取一个元素（默认步长）。
    • 步长 2：每次跳过一个元素，取间隔为2的元素。
