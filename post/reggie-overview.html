瑞吉外卖系统功能与问题记录

一、核心功能说明

  1. 数据排序与查询
    系统支持两种核心排序方式：

    • 按更新时间排序：适用于动态数据展示（如订单列表、菜品更新记录），默认展示最新数据在前
    • 按用户姓名排序：多用于用户管理模块，支持字母 / 拼音升序 / 降序切换，便于快速定位特定用户

    查询功能支持实时交互：

    • 输入框动态更新：用户输入关键词时，实时匹配相关结果（如菜品名称、用户名），无需等待点击查询按钮
    • 回车快捷查询：输入完成后按回车键可直接触发查询，提升操作效率

  2. 界面交互设计

    • 左侧菜单栏：支持收起 / 展开切换，收起时仅显示图标，节省页面空间，适合大屏数据展示场景
    • 状态切换控件：采用滑动按钮实现 “启用 / 禁用” 状态切换（如菜品上下架、用户账号状态），视觉反馈清晰，操作比传统开关更直观
    • 权限与安全控制：
      • 密码安全：强制密码复杂度校验（含大小写字母、数字、特殊字符），支持定期密码重置提醒
      • 访问控制：可配置 “游客访问” 权限，限制未登录用户仅能浏览菜品列表，无法下单或查看个人信息

  3. 批量操作与数据管理

    • 批量处理功能：支持用户、菜品等数据的批量删除、状态修改，操作前需勾选目标项
    • 时间范围控制：可设置数据禁用的时间范围（如特定活动菜品的下架时段），到期后自动恢复状态
    • 全选与撤销：列表顶部提供 “全选” 按钮，勾选后可一键取消选择，简化多数据操作流程

二、已知问题与场景复现

  1. 批量删除异常
    • 问题：当未勾选任何数据时，点击 “批量删除” 按钮，界面会弹出错误提示（未处理空选择场景）
    • 影响：影响用户操作体验，需优化前端校验，禁用空选择时的删除按钮
  2. 菜品添加流程缺陷
    • 场景：添加菜品时，未填写任何菜品信息，直接点击 “取消” 或 “确定” 按钮后，再次点击 “添加菜品”，之前暂存的菜品数据会丢失；若此时点击 “确定”，会导致空数据提交
    • 建议：增加本地缓存暂存功能，或在取消时提示 “数据未保存”
  3. 地址管理问题
    • 设为默认地址后，无法取消勾选：默认地址仅支持切换，不支持取消 “默认” 状态（至少保留一个默认地址），需优化交互说明
    • 结算依赖默认地址：用户必须设置默认地址才能进入 “去结算” 流程，未设置时应引导跳转至地址管理页面
  4. 购物车与勾选状态问题
    • 忌口选项强制勾选：购物车中 “忌口” 选项为必选，且无法取消勾选，不符合部分用户无特殊要求的场景
    • 页面跳转后勾选状态丢失：在购物车勾选商品数量后，跳转至其他页面（如地址页）再返回，之前的勾选数量会重置
