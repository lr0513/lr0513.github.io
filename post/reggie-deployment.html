1. 目录内容查看

  • 命令: ls
  • 语法: ls [-al] [dir]
  • 说明:
    • a: 显示所有文件及目录（包括隐藏文件，以.开头）。
    • l: 显示详细信息，包括文件类型、权限、拥有者、文件大小等。
  • 示例:
    • ls -al等价于ll: 查看当前目录所有文件及目录详细信息。
    • ls -al /etc: 查看/etc目录下所有文件及目录详细信息。

2. 切换目录

  • 命令: cd
  • 语法: cd [dirName]
  • 特殊说明:
    • ~: 用户的 home 目录。
    • .: 表示当前目录。
    • ..: 表示当前目录的上级目录。
  • 示例:

    • cd ..: 切换到当前目录的上级目录。

    • cd ~: 切换到用户的 home 目录。

    • cd /usr/local: 切换到/usr/local目录。

    • 注：用户的home目录

      ——root用户 /root

      ——其他用户 /home/xxx

3. 创建文件

  • 命令: touch
  • 语法: touch [文件名]
  • 示例:
    • touch 1.txt: 创建文件1.txt。
    • touch 2.txt 3.txt 4.txt: 一次性创建文件2.txt, 3.txt, 4.txt。

4. 创建目录

  • 命令: mkdir

  • 语法: mkdir [-p] dirName

  • 说明:

    • p: 确保目录名称存在，不存在则创建。可实现多层目录同时创建。

  • 示例:

    • mkdir itcast: 在当前目录下创建名为itcast的子目录。
    • mkdir -p itcast/test: 在itcast目录中创建名为test的子目录，若itcast目录不存在，则建立。

    rmdir

    • 作用：删除空目录。
    • 语法：rmdir [-p] dirName
    • 说明：
      • p：如果父目录在删除子目录后变为空目录，那么也一并删除父目录。
    • 示例：
      • rmdir itcast：删除名为 itcast 的空目录。
      • rmdir -p itcast/test：删除 itcast 目录中名为 test 的子目录，如果 test 目录被删除后 itcast 目录变为空目录，则 itcast 目录也会被删除。
      • rmdir itcast*：删除所有以 itcast 开头的空目录。
      • rmdir *itcast：删除所有以 itcast 结尾的空目录。

5. 删除文件或目录

  • 命令: rm
  • 语法: rm [-rf] name
  • 说明:
    • r: 递归删除目录及其中的文件和子目录。
    • f: 无需确认，直接删除。
  • 示例:
    • rm -r itcast/: 删除名为itcast的目录及其中的所有文件和子目录，删除前需确认。
    • rm -rf itcast/: 无需确认，直接删除名为itcast的目录及其中的所有文件和子目录。
    • rm -f hello.txt: 无需确认，直接删除hello.txt文件。

6. 拷贝文件或目录

  • 命令: cp
  • 语法: cp [-r] source dest
  • 说明:
    • r: 如果复制的是目录，将复制该目录下所有的子目录和文件。
  • 示例:
    • cp hello.txt itcast/：将 hello.txt 复制到 itcast 目录中。
    • cp hello.txt ./hi.txt：将 hello.txt 复制到当前目录，并改名为 hi.txt。
    • cp -r itcast/ ./itheima/：将 itcast 目录及其下所有文件复制到 itheima 目录下。
    • cp -r itcast/* ./itheima/：将 itcast 目录下所有文件复制到 itheima 目录下。

7. 移动或重命名

  • 命令: mv

  • 语法: mv source dest

  • 示例:

    • mv hello.txt hi.txt: 将hello.txt重命名为hi.txt。
    • mv hi.txt itheima/: 将文件hi.txt移动到itheima目录中。
    • mv hi.txt itheima/hello.txt：将 hi.txt 移动到 itheima 目录中，并重命名为 hello.txt。
    • mv itcast/ itheima/：如果 itheima 目录不存在，将 itcast 目录重命名为 itheima。
    • mv itcast/ itheima/：如果 itheima 目录存在，将 itcast 目录移动到 itheima 目录中。

    mv 命令既能够改名，又可以移动，具体是改名还是移动，系统会根据我们输入的参数进行判定(如果第二个参数dest是一个已存在的目录,将执行移动操作,其他情况都是改名)

8. 打包压缩

  • 命令: tar

  • 语法: tar [-zcxvf] fileName [files]

  • 后缀说明：

    • .tar：仅打包，不压缩。
    • .tar.gz 或 .tgz：打包并使用gzip进行压缩。

  • 说明:

    • z: 使用gzip压缩或解压。
    • c: create创建新的包文件。
    • x: 从包文件中还原extract文件。
    • v: 显示命令执行verbose过程。
    • f: 指定包文件file的名称。

  • 示例:

    • 打包：
      • tar -cvf hello.tar ./*：将当前目录下所有文件打包，文件名为hello.tar。
      • tar -zcvf hello.tar.gz ./*：将当前目录下所有文件打包并压缩，文件名为hello.tar.gz。
    • 解包：
      • tar -xvf hello.tar：解包hello.tar文件，并将文件放在当前目录。
      • tar -zxvf hello.tar.gz：解压hello.tar.gz文件，并将文件放在当前目录。
      • tar -zxvf hello.tar.gz -C /usr/local：解压hello.tar.gz文件，并将文件放在/usr/local目录。

    注：解压到指定目录,需要加上参数 -C

9. 文本编辑

  • 命令: vim
  • 语法: vim fileName

![[c126c05807d74bd19147f9d50adbef24.png]]

  • 命令模式（Command mode）：用于执行命令，如移动光标、删除文本等。

    • 打开 Vim，默认进入命令模式。
    • 指令：
      • gg：定位到文件的第一行。
      • G：定位到文件的最后一行。
      • dd：删除光标所在行。
      • ndd：删除当前行及之后 n 行。
      • u：撤销操作。
      • shift+zz：保存并退出。

  • 插入模式（Insert mode）：用于插入文本。

    • 按 i、a 或 o 进入插入模式。
    • 按 ESC 键从插入模式返回命令模式。

  • 底行模式（Last line mode）：用于执行更复杂的命令，如保存文件、退出编辑器等。

    • 按 : 进入底行模式。
    • :wq：保存并退出。
    • :q!：不保存退出。
    • :set nu：显示行号。
    • :set nonu：取消行号显示。
    • :n：定位到第 n 行，例如 :10 定位到第 10 行。

  • 如果文件存在，使用 vim 文件名 打开并编辑文件。

  • 如果文件不存在，使用 vim 文件名 创建并编辑新文件。

cat:

  • 作用：cat 命令用于显示文件内容。
  • 语法：cat [-n] fileName

more:

  • 语法：more fileName
  • 操作说明：
    • 回车键：向下滚动一行。
    • 空格键：向下滚动一屏。

tail:

  • 作用：tail 命令用于查看文件末尾的内容。
  • 语法：tail [-f] fileName
  • 选项：
    • f：跟随文件，动态读取文件末尾内容并显示。当文件更新时，tail 会实时显示新增的内容，通常用于实时查看日志文件的更新。
    • n：后面可以跟一个数字，表示显示文件末尾的行数。如果不指定 n，则默认显示最后10行。
  • 示例：
    • tail /etc/profile：显示 /etc 目录下的 profile 文件末尾10行的内容。
    • tail -20 /etc/profile：显示 /etc 目录下的 profile 文件末尾20行的内容。
    • tail -f /itcast/my.log：动态读取 /itcast 目录下的 my.log 文件末尾内容并显示。

1.3.3 防火墙操作
前面我们已经通过日志的方式及查看系统进程的方式，验证了Tomcat服务已经正常启动，接下来我们就可以尝试访问一下。访问地址：

![[af058263045f445b99dd837f2216b8a4.png]]

那为什么tomcat启动成功了，但就是访问不到呢？原因就在于Linux系统的防火墙，系统安装完毕后，系统启动时，防火墙自动启动，防火墙拦截了所有端口的访问。接下来我们就需要学习一下，如何操作防火墙，具体指令如下：

  1. 查看防火墙状态：
    • 使用命令：systemctl status firewalld 或者 firewall-cmd --state
    • Ubuntu：sudo ufw status
    • 备注：可以查看防火墙是否正在运行。active/inactive
  2. 暂时关闭防火墙：
    • 使用命令：systemctl stop firewalld
    • U:sudo ufw enable
    • 备注：这将立即停止防火墙服务，但不会影响系统的下次启动。
  3. 永久关闭防火墙（禁用开机自启）：
    • 使用命令：systemctl disable firewalld
    • U:sudo ufw disable
    • 备注：这将关闭防火墙服务，并设置为下次启动时不自动启动。
  4. 暂时开启防火墙：
    • 使用命令：systemctl start firewalld
    • 备注：这将立即启动防火墙服务。
  5. 永久开启防火墙（启用开机自启）：
    • 使用命令：systemctl enable firewalld
    • 备注：这将设置防火墙服务为下次启动时自动启动。
  6. 开放指定端口：
    • 使用命令：firewall-cmd --zone=public --add-port=8080/tcp --permanent
    • 备注：这将开放端口8080，-permanent标志表示永久生效，需要执行重新加载。
  7. 关闭指定端口：
    • 使用命令：firewall-cmd --zone=public --remove-port=8080/tcp --permanent
    • 备注：这将关闭端口8080，-permanent标志表示永久生效，需要执行重新加载。
  8. 立即生效（重新加载）：
    • 使用命令：firewall-cmd --reload
    • 备注：应用当前规则库的更改，不重启服务。
  9. 查看开放端口：
    • 使用命令：firewall-cmd --zone=public --list-ports
    • 备注：列出当前区域下开放的所有端口。

分布式VS集中式

除了git还有svn、cvs这样的版本控制系统，它们的区别在于一个是分布式一个是集中式

集中式就是svn和csv这样的版本控制系统，分布式是git

区别在于集中式的版本控制系统每次在写代码时都需要从服务器中拉取一份下来，并且如果服务器丢失了，那么所有的就都丢失了，你本机客户端仅保存当前的版本信息，换句话说，集中式就是把代码放在一个服务器上集中管理，你的所有回滚等操作都需要服务器的支持。

分布式的区别在于，每个人的电脑都是服务器，当你从主仓库拉取一份代码下来后，你的电脑就是服务器，无需担心主仓库被删或者找不到的情况，你可以自由在本地回滚，提交，当你想把自己的代码提交到主仓库时，只需要合并推送到主仓库就可以了，同时你可以把自己的代码新建一份仓库分享给其它人。

像集中式它们都有一个主版本号，所有的版本迭代都以这个版本号为主，而分布式因为每个客户端都是服务器，git没有固定的版本号，但是有一个由哈希算法算出的id，用来回滚用的，同时也有一个master仓库，这个仓库是一切分支仓库的主仓库，我们可以推送提交到master并合并到主仓库上，主仓库的版本号会迭代一次，我们客户端上的git版本号无论迭代多少次，都跟master无关，只有合并时，master才会迭代一次。

远程仓库常用命令

  • git remote——查看远程仓库

    列出所有远程仓库：git remote -v

  • git remote add ——添加远程仓库 shortname一般是origin

  • git remote set-url ——更改远程仓库的 URL

  • git remote remove ——删除远程仓库

  • git clone——从远程仓库克隆

  • git pull ——从远程仓库拉取

  • git push ——推送到远程仓库

要先提交到本地仓库，再提交到远程仓库

基本步骤：

  1. 在本地进行更改：对项目中的文件进行修改或添加新文件。【文件修改之后也要先在本地add】

  2. 暂存更改：使用 git add 命令将更改的文件添加到暂存区

  3. 提交更改到本地仓库：使用 git commit 命令将暂存区的更改提交到本地仓库。

    若提示Fatal: cannot do a partial commit during a merge.则在最后添加-i

  4. 推送更改到远程仓库：使用 git push 命令将本地仓库的更改推送到远程仓库。

    其中 <branch_name> 是你想要推送的分支名，例如 master 或 main。

    git push origin master

  5. 检查远程仓库：（可选）有时你可能想要先检查远程仓库的状态，然后再推送，可以使用 git fetch 命令来获取远程仓库的最新状态而不自动合并。

    这将从远程仓库获取所有分支的最新状态，但不会自动合并到当前分支。

    git fetch origin

  6. 解决冲突（如果有的话）：如果在推送过程中出现冲突（比如远程分支有你本地没有的提交），Git 会要求你解决这些冲突。解决完冲突后，再次提交并推送。

  7. 拉取远程更改：如果你想要获取远程仓库的最新更改并合并到你的本地分支，可以使用 git pull 命令。

    这会拉取远程分支的最新更改并与你的本地分支合并。

    注意：如果当前本地仓库不是从远程仓库克隆，而是本地创建的仓库，并且仓库中存在文件，此时再从远程仓库拉取文件的时候会报错(fatal:refusing to merge unrelated histories)

    解决此问题可以在qit pull命令后加入参数–allow-unrelated-histories

分支操作

分支是Git 使用过程中非常重要的概念。使用分支意味着你可以把你的工作从开发主线上分离开来，以免影响开发主线。

同一个仓库可以有多个分支【是一种动态概念】，各个分支相互独立，互不干扰。

通过git init 命令创建本地仓库时默认会创建一个master分支。

master：master 通常是一个默认的分支名，它代表主分支。当你首次初始化一个新的 Git 仓库时，Git 会自动创建一个名为 master 的分支，并且你的第一次提交将会在这个分支上。
